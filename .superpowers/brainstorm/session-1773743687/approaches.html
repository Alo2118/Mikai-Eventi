3 Approcci Architetturali

Clicca sull'approccio che preferisci

A

React SPA + Supabase

Frontend: React + Vite + TailwindCSS → GitHub Pages

Backend: Supabase (PostgreSQL + Auth + Realtime + Storage)

Mobile: PWA (installabile da browser, offline-capable)

Pro

  • Zero costi — tutto free tier
  • Supabase gestisce auth, DB, notifiche push, storage file
  • Realtime integrato (liste aggiornate live)
  • PWA funziona come app nativa su mobile
  • Row Level Security per permessi ufficio vs commerciali

Contro

  • Dipendenza da servizio esterno (Supabase)
  • Free tier: DB pausa dopo 1 settimana inattività (si riattiva)
  • React ha curva di apprendimento se dovete manutenere

⭐ RACCOMANDATO — miglior rapporto funzionalità/costo/semplicità

B

Next.js Full-Stack + Supabase

Frontend + API: Next.js (SSR) → Vercel free tier

Backend: Supabase per DB e Auth

Mobile: PWA

Pro

  • Server-side rendering (SEO, performance)
  • API routes integrate nel progetto
  • Vercel free tier generoso

Contro

  • Più complesso da sviluppare e deployare
  • Vercel free ha limiti su serverless functions
  • Over-engineering per un'app interna aziendale
  • Non gira su GitHub Pages (serve Vercel)
C

Vue.js + Firebase

Frontend: Vue 3 + Vite → GitHub Pages

Backend: Firebase (Firestore + Auth + Cloud Messaging)

Mobile: PWA

Pro

  • Vue più semplice da imparare di React
  • Firebase ha push notifications native
  • Free tier generoso (Spark plan)

Contro

  • Firestore è NoSQL — meno adatto per query relazionali (eventi↔materiale↔staff)
  • Costi imprevedibili se superi il free tier
  • Vendor lock-in Google più forte
